# Xart

Xart - это современное веб-приложение для просмотра аниме, состоящее из фронтенд-части на Vue.js и бэкенд-части на Go. Приложение предоставляет удобный интерфейс для поиска, просмотра и отслеживания аниме и манги с возможностью авторизации и персонализации.

## Архитектура проекта

Проект состоит из двух основных частей:

### Frontend
- **Фреймворк**: Vue 3 с использованием Composition API
- **Язык программирования**: TypeScript
- **Стили**: Tailwind CSS
- **Маршрутизация**: Vue Router
- **Управление состоянием**: Pinia
- **Сборка**: Vite
- **Иконки**: Используются Iconify иконки
- **Дополнительные библиотеки**: CropperJS, Markdown-it, Swiper

### Backend
- **Язык программирования**: Go
- **Версия Go**: 1.25.1
- **Тип приложения**: HTTP-сервер
- **Версия приложения**: 3.9.2
- **Прокси-сервер**: Проксирует запросы к API anixart.com
- **Функциональность**: Загрузка информации о версиях, проксирование API-запросов

## Установка и запуск

### Frontend

1. Установите зависимости:
```bash
cd frontend
npm install
```

2. Запустите режим разработки:
```bash
npm run dev
```

3. Для сборки продакшн-версии:
```bash
npm run build
```

### Backend

1. Убедитесь, что у вас установлена Go версия 1.25.1 или выше

2. Запустите сервер:
```bash
cd backend
go run main.go
```

3. По умолчанию сервер запускается на порту 8080

### Переменные окружения

Backend поддерживает следующие переменные окружения:

- `PORT` - порт, на котором будет запущен сервер (по умолчанию 8080)
- `API_URL` - URL внешнего API (по умолчанию https://api-s.anixsekai.com)
- `CHANGELOG_DIR` - директория с файлами изменений (по умолчанию ../frontend/public/changelog)
- `CORS_ORIGIN` - разрешенные источники для CORS (по умолчанию *)

## API

Backend предоставляет следующие эндпоинты:

- `GET /healthz` - проверка статуса сервера
- `GET /api/version` - информация о текущей версии приложения
- `GET /api/version/{version}` - информация о конкретной версии
- `GET/POST /api/proxy/*` - прокси-запросы к внешнему API

## Скрипты запуска

Для удобства разработки в проекте есть скрипты для запуска в режиме разработки:

- `run-dev.cmd` - командный файл для Windows
- `run-dev.ps1` - PowerShell-скрипт для Windows
